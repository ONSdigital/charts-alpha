User need

  • I need to know if there is other content related to what I’m reading so that I can get a full picture.
  • I need to know when something was released, so that I know it's relevant
  • I need to know what the content is before I click it

Prototype

  • Current website example
  • Prototype

Related links box in sidebar

  • Only one place in the sidebar for related links
  • Less visual noise
  • Consistent background colour
  • More generious whitepace
  • Formalise related links in content area

Related links section

  • Formalise related links in content area
  • Each link has title, meta and a description
  • Meta could include release date, content type and whether the link is external or not

Questions for research

  • Do users use links in content section more than sidebar?
  • Would users benefit from having fewer, more popular links in the sidebar?
  • Do users understand the difference between 'Related publications', 'You might also be interested in:' and 'Links to related statistics'
  • Do users need extra information (e.g. release date, description, content type?)
  • Do users need a warning if the link is going to take them to a an external site?

Context

ONS website

  • List of highlighted content in intro box on landing page
  • List of related publications on product pages
  • Related links in sidebar
  • Related links at bottom of page
  • You might also be interested in
  • Variations of this time series

Visual blog

  • List of related links in sidebar under heading: related
  • Sometimes the description is used as a content label: e.g. Bulletin or interactive
  • Sometimes the description descrive the link content e.g. Landing page for preliminary, second and final estimates of GDP

Future areas for exploration

  • Tags. What do they look like? Where will they live? Don’t call it tagging, call it something like related topics. This should be in addition to taxonomic structure. Related topics bar at bottom of page to allow cross navigation. Will users click it? Let’s A/B test this.
  • Tag product page. Where does it take you? What will it look like? A product page? There are lots of “list” pages and they all look and work differently, is this right?
  • Collections. These could work for tags, releases and even “theme days”. Collections on GOV.UK
